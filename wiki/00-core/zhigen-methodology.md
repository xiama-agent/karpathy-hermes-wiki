# 治本方法论

> 注：本页记录「治本」一词在龙虾语境中的精确定义。YANG 使用这个词时有特定的含义，不可偏离。

## 定义

**治本 = 找到问题的根本原因 + 解决根本原因**

不做表面的修修补补，不堆临时方案。找到问题真正出在哪一层，然后在那层解决它。

## 与卡帕西框架的关系

卡帕西第二大脑框架（SOUL.md + AGENTS.md + Wiki + Skill）是**做治本的底层基础**，不是治本的定义。

- 卡帕西框架 = 承载修复方案的地方（写进 Wiki、Skill、宪法）
- 治本 = 找根因的方法论

两者不冲突。治本找出来的修复方案，**用卡帕西框架来承载**。

## 治本 vs 治标的判断标准

| 治标 | 治本 |
|------|------|
| 加一个新脚本/工具 | 在 Wiki/Skill/SOUL.md 中记录流程 |
| 临时手动操作一次 | 写成可复用的 checklist 或 skill |
| 绕过问题（换条路） | 找到为什么需要绕过，修底层原因 |
| 加一条临时规则 | 在宪法层面加不可违背的原则 |

## 当前对话中已应用的治本案例

1. 视觉不能看图 → 根因：vision_analyze 不支持 MiMo → 治本：SOUL.md 第10条 + Wiki 记录 MiMo 用法
2. 派单不准 → 根因：龙虾概括原文导致失真 → 治本：SOUL.md 第11条 + Reasonix 介入拆解
