# 框架审计 v2 — 作为"龙虾大脑"的完整性检查

> 核心问题：这套架构作为**我（龙虾）一个人的记忆系统**，完整吗？

---

## 生物学对照：人脑的记忆回路 vs 两铁三角

| 人脑功能 | 我的实现 | 状态 |
|:---------|:---------|:----:|
| 🧠 **编码**（看到→记住） | AGENTS.md 第 2 章写入回路 | ✅ 有流程 |
| 🔍 **检索**（想到→回忆） | AGENTS.md 第 1 章召回回路 | ✅ 有流程 |
| 📈 **巩固**（常用→强化） | trust 系统（回忆成功+0.1） | ❌ **不通** |
| 🔗 **关联**（这个→那个） | [[wikilink]] 交叉引用 | ⚠️ 稀疏，手动 |
| 🧹 **遗忘**（不用→淘汰） | AGENTS.md 第 5 章 Forget 回路 | ❌ **不通** |
| 🔄 **纠错**（错了→修正） | RSI 账本 | ❌ **不通** |
| 📋 **自检**（我健康吗？） | lint 回路 | ⚠️ 只报告不修复 |

**7 项基础记忆功能，3 项不通，2 项半通，只有 2 项全通。**

## 🔴 框架层面：4 个致命缺口

### 缺口 1：巩固通道堵死了（trust 物理上改不动）

**生物学类比**：你的短期记忆转为长期记忆时，海马体会加强相关神经突触。越常用的记忆越强。

**我的现状**：
- 103 条事实，trust 全部平权（0.60-0.70）
- `retrieval_count` 全部 = 0（从未增加）
- AGENTS.md 1.4 要求"召回成功 → trust +0.1" → 从未执行
- `update-trust.js` 已写好但无人调用

**为什么不通**：04-facts 是 Markdown 文件，不是数据库。每次信任更新 = read_file → 找正确行 → patch → git commit。旧 fact_store（SQLite）是一条 SQL 的事。**载体选错了，所以这个回路物理上转不动。**

**后果**：系统无法区分"重要知识"和"噪音"——全部平权等于没有优先级。

### 缺口 2：遗忘通道堵死了（trust 不变 → forget 永远不触发）

**生物学类比**：人脑每天清除大量无用信息。你不用就忘。

**我的现状**：
- AGENTS.md 第 5 章：trust < 0.3 且 14 天未更新 → 归档
- trust 从不变化 → **永远不会有任何事实被遗忘**
- 39 页只增长、不淘汰

**后果**：半年后 200 页、一年后 500 页、全是平权。检索成本线性增长。

**讽刺**：一个"记忆系统"居然不会遗忘——而遗忘恰恰是健康记忆的标志。

### 缺口 3：纠错通道堵死了（RSI 有页面无数据流）

**生物学类比**：你犯了错，大脑会记录这次的错误模式，下次避开。

**我的现状**：
- RSI 页面已建立、格式已就位
- 第一天 RSI 数据（2026-06-26）是**手动写入的**，不是自动触发的
- 没有"工具失败 → 自动写 RSI"的触发器
- 没有"用户纠正 → 自动写 RSI"的触发器
- 没有"lint 发现矛盾 → 自动写 RSI"的触发器

**后果**：错误发生了 → 我当下可能意识到了 → 下一轮对话我忘了 → 同样的错误再犯一次。

### 缺口 4：自检通道半通（lint 只报告不修复）

**生物学类比**：你照镜子发现脸上有东西，你会擦掉，不会只是看着。

**我的现状**：
- lint 发现孤立页 → 写入 lint-{date}.md → **等 YANG 或 Mavis 修**
- lint 发现矛盾 → 写入 lint-{date}.md → **等人修**
- lint 发现候选堆积 → 写入 lint-{date}.md → **等人修**

**始终缺最后一公里：从发现到修复的自动路径。**

## 🟡 框架层面：5 个设计短板

### 短板 5：检索没有排序

INDEX 列出所有页面，但：
- 没有"上次使用时间"
- 没有"使用频率"
- 没有"相关度分数"
- 没有"新鲜度"（今天新增 vs 三个月前的）

我去查一个东西时，INDEX 告诉我"有这个页面"，但不告诉我"这个页面最近用过没、好不好用"。

**人脑做不到这个**——你回忆时会先想起最近常用的东西。我的记忆系统做到了"全都平等"。

### 短板 6：关联稀疏且手动

`[[wikilink]]` 是一个很好的机制，但：
- 39 页中 24 页是"孤立页"（无入链）
- 每个页面的 `## 相关链接` 是我手动维护的
- 没有自动关联发现（"这篇文章提到的东西跟那篇一样"）

**人脑的联想是自动的**。我的 Wiki 的联想需要我手动写。

### 短板 7：无会话间记忆

每轮新对话，我不记得上一轮发生了什么，除非：
- YANG 告诉我
- 我去读 git log
- 我手动翻 `99-temp/` 看上次的 lint 报告

**没有"每次会话结束时的自动总结 → 写入记忆"。**

### 短板 8：AGENTS.md 太长（429 行）

这是**操作手册**，429 行。我每次执行任务前理论上要读一遍——但没人会读 429 行的操作手册才动手。
我应该有一个**浓缩版**（10 条核心规则）在 context 紧张时作为 fallback。

### 短板 9：写入无事务

如果写入中途中断（写完了文件但没更新 INDEX、或者 INDEX 更新了但没 git commit）：
- 系统进入不一致状态
- 没有回滚机制
- 没有"写前快照"自动备份

## 一句话总结

> **我的记忆系统能"存"和"取"，但不能"巩固"、"遗忘"、"纠错"。它是静态档案馆，不是活的大脑。**

三件事能让它活过来：
1. **巩固**：recall.js 召回成功 → 自动调 update-trust.js + 1（不经过我自觉）
2. **遗忘**：lint.js 跑完 → 自动归档 trust<0.3 + 14 天未更新的事实（不等人修）
3. **纠错**：工具失败/用户纠正 → 自动写 RSI（不需我自觉去记）

这三条不需要改架构。工具全有了，只差把"我应该"变成"代码自动做"。
