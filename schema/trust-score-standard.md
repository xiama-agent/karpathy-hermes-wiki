# Schema: Wiki 页面信任分规范

> 2026-06-27 建立。定义所有 Wiki 页面的 frontmatter 信任分标准。
> 对应 SOUL.md v4.0 卡帕西架构中的 `schema/` 约束层。

## 信任分规则

| 分值 | 含义 | 适用页面 |
|:----:|------|---------|
| **1.0** | 不可变事实 | 铁律、偏好、路径、身份定义 |
| **0.95** | 已验证配置 | 工具路径、API 接入、cron 任务 |
| **0.9** | 高置信度知识 | 项目索引、工作流、已验证的集成方案 |
| **0.7** | 常规事实 | 用户偏好、通用知识 |
| **0.65** | 中等置信 | 配置参考、功能说明 |
| **0.6** | 低置信 | 待验证、待确认的信息 |
| **0.3** | 暂存/草稿 | 临时笔记、未验证的架构分析、初稿 |
| **0.1** | 归档/历史 | 已归档的旧记录、一次性任务报告 |

## 不要求信任分的文件

- `98-archive/session-log/*.md` — 会话日志，自动生成，不维护信任分
- `99-temp/*.md` — 临时候选，不维护信任分
- `log/*.md` — 运行日志，不维护信任分

## 检查命令

```bash
# 扫描信任分缺失
for f in wiki/00-core/*.md wiki/01-yang/*.md wiki/02-knowledge/*.md wiki/03-system/*.md wiki/04-facts/*.md; do
  [ -f "$f" ] && grep -q "^trust:" "$f" || echo "❌ $f"
done

# 统计信任分分布
grep -h "^trust:" wiki/*/*.md 2>/dev/null | sort | uniq -c | sort -rn
```
